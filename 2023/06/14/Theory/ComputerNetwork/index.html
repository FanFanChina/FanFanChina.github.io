域名（Domain）

域名的由来

域名的出现源于对互联网地址的需求，由美国计算机科学家保罗·穆克鲁斯（Paul Mockapetris）于1983年首次提出。

在当时的ARPANET（美国国防部资助的一个计算机网络）中，每个计算机都使用一个唯一的数字 IP 地址来标识自己。然而，这些 IP 地址十分复杂，不方便用户记忆和使用，也不具备任何意义。

为此，保罗·穆克鲁斯设计了域名系统（DNS），将每个 IP 地址与一个更加易于记忆和使用的字符串相对应，从而方便用户访问特定的资源。最初，域名仅包含少量的顶级域名，如 .com、.edu、.org 等，后来随着互联网的发展，新增了许多二级域名和三级域名等。

现在，域名成为互联网的重要组成部分，并且被广泛应用于各种网络服务和应用场景中。它不仅为用户提供了方便和快捷的访问方式，也为企业、组织和个人提供了独立的品牌形象和在线服务。

域名分类

  1. 顶级域名（Top-Level Domain，TLD）：顶级域名是最高层次的域名，在互联网命名体系中是唯一的。它通常由少数几个字母组成，用于表示国家、地区、类型和组织等不同的分类。例如，.com、.cn、.org、.net 等就是常见的顶级域名。
  2. 二级域名（Second-Level Domain，SLD）：二级域名是在顶级域名下面的第二个层次的域名，通常用于表示特定的主机、企业或组织等。例如，在 baidu.com 中，“baidu” 就是一个二级域名，用于标识百度公司的网站。
  3. 子域名（Subdomain）：子域名是在二级域名或更高级别域名的基础上增加的额外标识，用于划分不同部门、项目或服务等。例如，在 mail.google.com 中，“mail” 就是一个子域名，用于表示 Google 公司的邮件服务。
  4. 泛域名（Wildcard Domain）：泛域名是一种特殊的域名格式，用于匹配多个具有相似名称的域名。例如，*.example.com 可以匹配所有以 example.com 结尾的子域名，如 mail.example.com、news.example.com 等。
  5. 国际化域名（Internationalized Domain Name，IDN）：国际化域名是一种可以包含非 ASCII 字符和多语言字符的域名格式，用于支持全球范围内的多语言环境。例如，可以使用中文、日文、阿拉伯文等字符来表示域名，如：中国互联网资源中心的域名为 中互资助 cn 。

域名解析流程

  1. 浏览器缓存：当用户第一次访问一个网站时，浏览器会尝试在本地缓存中查找该域名对应的 IP 地址。如果能够找到，则直接使用本地缓存中的结果，否则会进入下一步。
  2. 本地Host 文件：如果本地缓存中没有找到相应的 IP 地址，浏览器会向操作系统发出请求，判断本地 Hosts 文件中是否有该域名对应的 IP 地址映射关系，如果有，则直接返回解析结果，否则会继续进行下一步。
  3. 操作系统缓存：如果本地 Host 文件中没有找到相应的 IP 地址，浏览器会向本地 DNS 服务发出请求，询问其是否有相应的 DNS 缓存。如果有对应的DNS缓存，则直接返回缓存中的结果，否则会继续进行下一步。
  4. 路由器缓存：如果操作系统中也没有找到相应的 DNS 缓存，浏览器会向本地网络中的路由器或网关发送请求，查询其是否有相应的 DNS 缓存。如果能够找到，则直接返回缓存中的结果，否则会继续进行下一步。
  5. ISP DNS 服务器：如果以上缓存都无法提供有效的结果，浏览器会向互联网服务提供商（ISP）的 DNS 服务器发送请求，查询其是否有相应的 DNS 记录。如果能够找到，则直接返回 DNS 服务器中的结果，否则会继续进行下一步。
  6. 根域名服务器：如果 ISP 的 DNS 服务器也无法提供有效的结果，浏览器会向根域名服务器发送请求，询问它是否知道该域名所对应的 IP 地址。根域名服务器会将请求转发给顶级域名服务器（如 .com、.org 等），并返回相应的 DNS 记录。
  7. 顶级域名服务器：顶级域名服务器会对收到的请求进行处理，并将请求转发到相应的权威域名服务器上。例如，如果请求的域名是 www.example.com，那么顶级域名服务器会将请求转发给 .com 域名的权威域名服务器。
  8. 权威域名服务器：权威域名服务器是负责管理特定域名的 DNS 服务器，它会对收到的请求进行处理，并返回相应的 DNS 记录，包括该域名对应的 IP 地址、MX 记录、TXT 记录等。
  9. 浏览器显示：最后，浏览器会接收到来自权威域名服务器的 DNS 记录，并在屏幕上显示请求的网页内容。

域名解析

  1. A类型（address）：添加三级域名前缀，然后将该域名指向一个IPv4地址
  2. CNAME类型（Canonical Name）：将一个域名指向一个规范的官方的域名
