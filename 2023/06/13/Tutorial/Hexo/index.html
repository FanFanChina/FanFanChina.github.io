本文阐述如何使用 Hexo + Github Pages搭建个人博客

本地安装 Hexo

  1. 官方文档Hexo
  2. 安装 Git、Nodejs
  3. 选着一个存放博客资源的文件夹在给文件夹下打开 command prompt 输入命令 init Hexo
  1. 安装 npm 包管理器
  1. 安装hexo的上传插件deploy-git
  1. 本地测试Hexo服务
  1. 选择一个自己喜欢的主题，如何配置见相应主题配置文档

Github Pages

  1. Github 提供了一个免费的静态页面托管服务 Github Pages
  2. Github Pages 其实是一个Github Repository, 仓库配置如下
  1. 为了避免每次hexo deploy到github时输入密码，在hexo站点配置中
  1. 配置本机与github的ssh免密连接（略）
  2. 站点和主题配置完毕以后，上传到github
  1. 打开网站 your_user_name.github.io 即可

自定义域名

一、购买域名，例如域名 xxx.com

二、配置域名解析服务

  • A 记录类型

    • Ping Github Pages 获取 IP

    • 主机记录：@（代表域名本身，也可用 blog 等个人三级域名）

    • 记录值：刚刚 Ping 到的 IP

    • 此种方法无法在 Github Pages 设置中开启 Https

  • CNAME 记录类型（推荐）

    • 主机记录： @（代表域名本身，也可用 blog 等个人三级域名）
    • 记录值：your_user_name.github.io
    • 配置 Github Pages
      • 打开 Github 找到自己的托管静态页面的仓库
      • 点击 settings -> pages -> custom domain 输入你的域名
      • 在本地博客文件 source 文件夹下添加无后缀的文件 CNAME，文件内容为自己的域名（这一步很重要，如果没有添加 CNAME 文件每一次 hexo d 上传后自定义域名配置会失效 !!!）
