OS基础

什么是OS？

01.OS是计算机硬件与应用之间的一层软件，它方便我们管理硬件，例如显存，并帮助我们高效使用硬件，如开多个终端（窗口）

02.管理的硬件：CPU、内存、终端、磁盘、文件等

计算模型的发展

01.图灵机：一个会做一道菜的厨师，执行单个固定程序

02.通用图灵机：一个能看懂菜谱的厨师，可以执行不同程序

03.计算机：冯诺依曼架构的发明，将程序（代码）和数据放到计算机内部的存储器内，在程序的控制下自动执行用户程序，例如cs + ip指令指向要执行的语句

打开电源发生了什么？

01.BIOS（Base Input And Output System）

位置——固化存放在主板上的一个ROM芯片内

功能——自检，确认计算机硬件能否满足运行的基本硬件条件，检测各种外设，然后将Bootloader加载到内存中，最终将CPU的控制权限交给软件

02.Bootloader （引导加载程序）

位置——位于硬盘的第一个主引导扇区，就是硬盘的第一个512字节存储空间，功能——完成一系列的功能并加载OS

03.OS

位置：活动分区内

进程与线程

CPU管理的直观想法

设PC初值

详述：设置好PC（CS + IP）的初值，然后不断地取址往下执行

问题：包含IO指令的话会耗费大量时间等待IO指令的完成，导致CPU效率极低

解决方案：用等待IO的时间去做别事情，完成后再回来干（并发）

Note：其实就是单道程序 ——>> 多道程序

如何设计？

方案一：修改PC的值即可

问题：来回切换执行的地址确实可以实现并发，但是可能不同程序之间的数据会相互影响，导致数据出错，所以不仅仅要修改PC的值，还要还原很多其他东西

解决方案：用一个东西记录程序切出去之间的样子和截至地址 —— PCB

进程的引入

原因：区分运行中的程序和静态的程序

例子：一本未看的书，和一本正在看的书

理解：为了区分运行中的程序，即正在进行的程序，进入CPU执行的程序，将其命名为进程，它的内容和静态程序相比较可能是他的一部分，还包含起止未知等等

多进程图像

01.计算机创建进程的开端

计算机启动之后最后执行main函数，创建一个进程 shell or win桌面，然后通过键入命令，或打开软件创建新的进程

02.如何查看进程（以win为例）

打开任务管理器

03.如何组织进程？

对每个程序建立进程控制PCB，对不同状态的PCB将其插入不同队列

进程状态：正在执行、等待执行、等待就绪（缺东西）

04.多进程如何切换？

第一步：将正在CPU中执行的程序保存为PCB然后插入等待队列

第二步：从就绪队列中取出一个PCB还原到CPU中执行

05.多进程之间的影响

问题：多个进程共同放在一个内存中，可能在读写时访问到别的进程的内容，导致程序出错

解决方案：映射，将不同进程同一地址映射到不同的物理地址中去

CPU调度策略

一、CPU调度目标

01.如何让进程满意？

答：尽快结束任务、用户操作尽快响应、系统内耗时间少

02.总原则

答：系统专注于任务执行，又能合理调配任务

二：各种调度算法

01.FCFS（First Come，First Served）

概述：先来先调度

问题：一个简单询问业务的人该怎么办？

解决方案：短作业优先

02.SJF （Short Job First）

概述：短的先执行

优点：平均周转时间最短

问题：响应时间过长怎么办？

解决方案：时间片轮转调度RR

03.RR（Round Robin）

概述：分给每一个进程一个时间片，来回循环执行多个进程

问题：时间片越长 —— 响应时间越长、时间片越短 —— 吞吐量小

三、响应时间和周转时间同时存在，怎么办？

直观想法：定义前台任务和后台任务两个队列，前台RR，后台SJF，前后台的切换定义优先级来解决

问题：即前台进程优先级高于后台进程，系统进程优先级会高于用户进程，特点就是不公平，容易产生饥饿。
