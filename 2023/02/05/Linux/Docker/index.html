Docker 学习笔记

发表于2023-02-05|更新于2023-12-26|Linux
|阅读量:

引言

Docker安装在服务器上，包含多个镜像，Docker可以把一个镜像分出多个相同的容器，容器可以看成一个个小的服务器

镜像和容器的关系可以看成为类和对象的关系

Docker迁移十分的简单！！！

安装

Docker 乌班图安装（尽量在Tumx里面运行，防止意外关闭终端后，工作进度丢失）

翻到 Install using the repository 复制安装代码即可

Docker Command

省去sudo

为了避免每次使用docker命令都需要加上sudo权限，可以将当前用户加入安装中自动创建的docker用户组

执行完此操作后，需要退出服务器，再重新登录回来，才可以省去sudo权限。

镜像命令

容器命令

容器导出的是他的模板镜像而非本身

docker export/import 与 docker save/load的区别
export/import会丢弃历史记录和元数据信息，仅保存容器当时的快照状态
save/load会保存完整记录，体积更大

为容器添加端口映射

  • 创建时添加：docker run -p 20000:22 -p 8000:8000 –name test -itd ubuntu:20.04
  • 已经生成的容器无法添加端口，可以先转为镜像，然后重新生成容器

引言

Docker Compose 是一种用于定义和运行多个 Docker 容器的工具，它可以帮助简化 Docker 应用程序的部署和管理

除了 up 和 down 命令之外，Docker Compose 还提供了其他许多有用的命令，包括：

  • ps: 列出当前正在运行的 Docker Compose 服务和容器。
  • logs: 显示指定服务或容器的日志输出。
  • build: 使用 Dockerfile 构建服务镜像。
  • restart: 重启指定的服务或容器。
  • exec: 在指定的容器上执行命令。
