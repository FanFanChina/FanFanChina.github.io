static 的注意事项 ⚠️

  • 静态方法只能访问静态变量和静态方法

继承

  • Java 只支持单继承，一个类只能继承一个直接父类
  • Java 不支持多继承、但是支持多层继承
  • Java 中所有的类都直接或者间接继承于 Object 类（当JVM识别到用户定义的类没有父类时，就会让其继承于 Object 类）
  • 子类只能访问父类中非私有的成员

多态中的调用

调用相同名称的成员变量时：

  • 编译看左边：javac 编译代码的时候，会看左边的父类中有没有这个变量，如果有，编译成功，如果没有编译失败
  • 运行看左边：java 运行代码的时候，实际获取的就是左边父类中成员变量的值

调用相同名称的成员方法时：

  • 编译看左边：javac 编译代码的时候，会看左边的父类中有没有这个方法，如果有，编译成功，如果没有编译失败
  • 运行看右边：java 运行代码的时候，实际获取的就是右边

调用成员变量时：子类继承了父类的成员变量 name 且自己也拥有一个成员变量 name。我们定义的实例化对象时，是由父类类型 Animal 变量 a 指向子类 Dog 的实例化对象，所以当我们用 a 去调用 name 变量时，只会选择由父类继承而来的 name 变量。

若父类 Animal 中不定义 name 变量，那么子类 Dog 中也不含有继承的父类 name 变量，自然也就无法调用（即使子类中也有名为 name 的变量，也不会去调用），所以编译时会报错。

不用多态时，若定义为 Dog 类型的变量 d 指向 Dog 的实例化对象，然后用 d 去调用 name 变量时，会先到 Doge 自己的成员变量中去寻找 name，若没有则在继承的变量中寻找 name

调用成员方法时：因为我们定义的实例化对象时，是由父类类型 Animal 变量 a 指向子类 Dog 的实例化对象，所以当用对象 a 去调用 show 方法时，只会选择由父类继承而来的 show方法，当恰恰因为子类重写了父类的 show 方法，父类 show 方法翻新了，所以最后从调用结果上看，反而执行的是子类的 show 方法。

就近原则：

  1. 父类和子类的同名变量没有覆盖的说法，所以父类类型的变量调用时同名变量时，优先选父类的

  2. 成员方法有覆盖的说法，覆盖后的成员方法优先级更高，所以就是调用成员方法时，如果覆盖了就调用覆盖后的方法

包 & final

概述：包就是文件夹、包名就是文件夹名称

命名规范：域名反写 + 包的功能（全部小写）

全类名：包名 + 类名（可用来区分不同包中的同名类）

导包相关：

  • 使用同一个包中的类时，不需要导包
  • 使用 java.lang 包中的类时，不需要导包
  • 其他情况都需要导包
  • 如果同时使用两个包中的同名类，需要用全类名

final 的一个例子就是，字符串的底层源码中，其实是用 private final 修饰了一个byte数组，该数组储存的就是每个字符的asII 码，于是 字符串一旦创建，地址就固定了，不能更改，private 关键字保证了外界无法修改该字符串的内容，string 源码中也不提供对应的 get set 方法

  • 接口不代表一类事物，接口就是一种规则，是对行为的抽象

接口与类的关系

  • 类和类的关系：继承关系，只能单继承，不能多继承，但是可以多层继承
  • 类和接口的关系：实现关系，可以单实现，也可以多实现，还可以在继承一个类的同时实现多个接口
    • 遇到多个接口中出现同名抽象方法的情况时，只会重写一次，但多个接口都可调用
  • 接口和接口的关系：继承关系，可以单继承，也可以多继承
    • 如果实现类实现了最下面的子接口，那么就需要重写继承的所有的抽象方法
