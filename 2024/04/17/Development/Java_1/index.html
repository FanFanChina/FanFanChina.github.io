运算符

  • 整数除法运算会下取整
  • 在代码中，如果有小数参与计算，结果有可能不精确

注：在计算机中，浮点数采用二进制表示，并使用有限的位数来表示小数部分。由于二进制无法精确地表示某些十进制小数，例如 0.1，在进行浮点数运算时，可能会出现舍入误差。这种舍入误差可能会在多次运算中累积，导致最终结果与预期的结果有一定的差距。这种情况下，我们称之为浮点数精度问题。

字符串参与加法运算

  • 当＂+”操作中出现字符串时，这个“+”是字符串连接符，而不是算术运算符了

  • 无论自增 or 自减运算符放到前面还是后面，效果都是实现自增 or 自减
  • 当携带自增 or 自减运算符的同时进行运算或输出时，运算符在后面的从本位开始，在前面的从下一位开始

类和对象

关系：类实例化后称之为对象

公共类和非公共类的关系

在Java中，同一个源文件可以包含多个类，但只能有一个公共类（public class），且公共类的类名必须与源文件的名称相同。其他非公共类可以在同一个源文件中定义，但它们的访问权限将受到限制，只能在同一个 package 中被访问

注：实际开发中建议一个 class 文件中有包含一个公共类即可，不用写非公共类！！！

类的封装

概述：对象代表什么，就得封装对应的数据，并提供数据对应的行为

例如：人关门，门是一个对象，关门这个行为要封装到门的 class 里面，而不是人的 class 里面

private 关键字

就近原则和 this 关键字

就近原则：当成员函数中的局部变量和成员变量重名时，成员函数中会优先调用自己的变量

this 关键字：通过给成员变量添加this关键字，就可以区分重名的成员变量和局部变量了

构造方法

注：如果我们自己没有写任何的构造方法，那么虚拟机会自动给我们加一个空参构造方法

标准 JavaBean

  1. 类名需要见名知意（驼峰命名法）
  2. 成员变量使用 private 修饰
  3. 提供至少两个构造方法
    1. 无参构造方法
    2. 带全部参数的构造方法
  4. 成员方法
    1. 提供每一个成员变量对应的 setXxx() / getXxx()
    2. 如果还有其他行为，也需要写上
