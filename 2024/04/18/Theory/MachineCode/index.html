为何选二进制？

大多数人类文明选择了十进制 (因为手指头有 10 根，最开始人们计数是靠手去数的)

但也有例外，玛雅人采用 20 进制服务于天文历法的计数，楔形文字采用 60 进制

计算机在发明以后，需要选用一种进制来储存数据

当时选有 10 进制的，3 进制的，8 进制的，但事实证明 2 进制才是最不容易出错的

其实电路中电压的高低，是电容上电荷的多少，是磁盘上的磁化程度，光盘上的蚀刻程度，光纤中发光二极管或者 “小灯泡” 的亮度 —— 而这些全都是（几乎）连续的物理量，并没有什么天然的 “只能取两个值” 的属性。

计算机选择 2 进制只是因为相对而言 2 进制更不容易出错

原码

来源：原码就是十进制数的二进制表现形式、正数符号位为 0、负数符号位为 1（以一个八位的机器作示例）

对于原码的正数而言进行加法运算时，向数轴正方向移动，运算过程符合加法运算逻辑

但是对于原码的负数而言进行加法运算时，向数轴反方向移动，运算过程与加法运算逻辑正好相反

为了解决原码表示的负数的运算问题，我们可以：

  • 维持数轴正方向的原码不变
  • 将数轴反方向的原码反过来（1,1111111 从 - 127 跑到 0 的位置）

这样操作后，在对负数调整后对应的原码进行加减运算时，便会向着正确的方向移动了

为了和原码有所区分，我们将改进过的这种原码称为反码

反码

来源：为了解决原码不能计算负数的问题而出现的

  • 正数的反码 = 正数的原码（因为数轴正方向没变）
  • 负数的反码 = 负数的原码符号位不变，数值位取反（因为数轴反方向反过来了）

反码的问题：

对反码有运算：1,1111111 + 0,0000001 == 0,0000000

但实际上 -0 + 1 == 1，而不是 -0 + 1 == +0

这是因为在反码中 1,1111111 和 0,0000000 都表示 0，因此在进行跨 0 运算时会在原地多移动一步，体现在运算结果上就是与实际结果有 1 步的误差

为了解决这个问题：

  • 默认只有 0,0000000 表示 0
  • 数轴反方向的反码从 -1 开始向后移动一位，空出来的 -1 位置补充为 1,1111111（原来的反码 -0）

这样就解决了反码在跨 0 运算时会产生误差为 1 的问题

为了和反码有所区分，我们将改进过的这种反码称为补码

补码

来源：为了解决反码在跨 0 运算时会产生误差为 1 的问题

  • 正数的反码 = 正数的原码（因为数轴正方向和原来一致）
  • 负数的反码 = 负数的原码符号位不变，数值位取反，然后再加 1（因为数轴反方向反了过来且后移并填补了一个 1,1111111）

简单而言：负数的反码 = 负数的反码 + 1

因为节省了一个 -0 的表示，所以负数的补码相比于正数的补码可以多表示一个数，这个数也是该机器可表示的最小负数

计算机中数的储存和计算都是以补码的形式进行的！！！

总结

对于一个 8 bit 的机器而言

能表示从 00000000 到 11111111 共 256 个编码

编码对十进制数的 3 种不同编排方式也就造就了原码、反码和补码三种编码方式

所以对于同一台机器而言，原码、反码和补码各自构成的集合之间是相等的

区别就在于，补码这种编排方式最终能符合我们对于计算的要求

而且补码相比于原码和反码还能多表示一位负数 😄
