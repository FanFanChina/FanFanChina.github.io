MathTheory

F_分解质因数

算术基本定理：每个正整数都能唯一分解为它的质因数的乘积 $N=P1^{a1} * P2^{a2} *P3^{a3}…..Pn^{an}$

  • Q1：循环内的 i 不一定是质数啊？
  • A1：假如 i 是一个合数，那么它一定可以分解成多个质因子相乘的形式，这多个质因子同时也是 a 的质因子且比 i 要小，而比 i 小的数在之前的循环过程中一定是被条件除完了的，所以 i 不可能是合数，只可能是质数，这也是循环从最小质数2不断增大的原因

K_快速幂

问题：快速计算 $a^k \bmod p$ 的大小

原理：反复平方法、二进制取幂

例子：计算 $7^9$

第一步：预处理出 $a^{2^0} \ a^{2^1} \ a^{2^2} \ … \ a^{2^{logk}}$ 的值（反复平方法）

  • $7^{(0001)_2} = 7$
  • $7^{(0010)_2} = 7^{(0001)_2} \times 7^{(0001)_2}$
  • $7^{(0100)_2} = 7^{(0010)_2} \times 7^{(0010)_2}$
  • $7^{(1000)_2} = 7^{(0100)_2} \times 7^{(0100)_2}$

第二步：拆分并组合

  • $7^9 = 7^{(1010)_2} = 7^{(1000)_2} \times 7^{(0010)_2}$

M_买不到的数目

前置知识

  1. 若 $gcd(p, q) = d$，那么 $ap + bq = kd$，即 $p$ 和 $q$ 能凑出 $d$ 的任何整数倍
  2. 若 $gcd(p, q) = d = 1$（$p$ 和 $q$ 互质）那么 $amp + bmq = m$，变形为 $(am + q)p + (bm - p)q = m$，可以看出如果 $m$ 足够大，终会使两系数化为正数
  • 规律：若 $gcd(p, q) = 1$ 那么 $ap + bq$ $(x≥0 \ y≥0)$ 不能凑出来的最大整数为 $(p - 1)(q - 1) - 1$

S_筛质数

朴素做法 $O(nlogn)$

  • 若 $p$ 没被筛掉，说明 $1 \sim p -1$ 都不能成为 $p$ 的约数，那么 $p$ 就一定是质数
  • 时间复杂度 $\dfrac{n}{2} + \dfrac{n}{3} + … + \dfrac{n}{n} = n(\dfrac{1}{2} + \dfrac{1}{3} + \dfrac{1}{n}) \approx n\ln n < nlog_2n \Rightarrow O(nlogn)$ (调和级数)

诶氏筛法 $O(n\log{\log n})$

  • 诶氏筛法其实是对朴素做法的优化
  • 质数定理：$1 \sim n$ 中存在 $\dfrac{n}{\ln n}$ 个质数

线性筛法 $O(n)$

S_试除法判定质数

  • 0 和 1 既不是质数也不是合数
  • 用试除法去判断 x 是否是质数的过程就是：试着用 x 去除 2 到 x - 1 的所有数，若可除整则为合数，否则为质数
  • 但实际上除数不用拓展到 x - 1，只用到 $\sqrt{x}$ 即可

Z_最大公约数

辗转相减法

$a$ 和 $b$ 的最大公约数表示为 $(a,b)$，那么 $p = (a, b) = (a, ca + b)$

注：如果 $p$ 是 $a$ 和 $ca+b$ 的公约数，$p$ 整除 $a$，也能整除 $ca+b$，那么就必定要整除 $b$

基于上面的原理，就能实现我们的迭代相减法（大数减小数）：

$(78, 14) \sim (64, 14) \sim (50, 14) \sim (36, 14) \sim (22, 14) \sim (8, 14) \sim (8, 6) \sim (2, 6) \sim (2,4) \sim (2, 2) \sim (2, 0)$

实际上，我们可以使用求余运算替代重复的减法运算，由此诞生了辗转相除法

辗转相除法

  • $(78,14) = (14 × 5 + 8,14) \sim (8,14)$
  • $(14, 8) = (8 + 6, 8) \sim (6, 8)$
  • $(8, 6) = (6 + 2, 6) \sim (2, 6)$
  • $(6,2) = (2×3,2) \sim(0,2)$
  • $(2, 0)$

计算出 $(78,14) = 2$

Z_组合数

一、公式法

$C_a^b = \dfrac{a(a - 1)\cdots(a - b+1)}{b(b - 1)\cdots1}$

二、递推法

$C_a^b = C_{a - 1}^b + C_{a - 1}^{b - 1}$

  • Q1：为什么递推法中子循环小于等于 $i - 1$
  • A1：因为递推法公式是 $C_a^b = C_{a - 1}^b + C_{a - 1}^{b - 1}$ 由于组合数的特性，下标永远大于上标，所以在 $C_{i - 1}^{j}$ 中 $j <= i - 1$

KMP字符串

  • Q1：为什么用next数组命名为ne而不是next？

  • A1：因为next可能会和某些库预制变量名称冲突

    ​

  • Q2：怎么求next数组？

  • A2：next[i] 就代表着 P[1 - i] 字符串的最大首位公共子串长度，既然是首尾比较得出next数组，那么我们可以通过 P字符串自己和自己匹配的过程，记录得出next数组的值

    ​

  • Q3：为什么求next数组时候循环从 2 开始，而KMP匹配时从 1 开始？

  • A3：因为求next数组时候 next[1] 一定等于 0，没必要从第一位开始比较，而且若真从第一位开始比较，必然使得next数组值呈现 1，2，3，4递增趋势，两个本就相同的模板串必然相互完全匹配，就得不到有效的next数组。KMP匹配时就存在开头就与模板串相同的情况，所以必须从 1开始。

    ​

  • Q4：两个循环里面为什么 j 都从 0 开始？

  • A4：这里的 j 代表的是成功匹配的位数，等于 0 表明还没开始匹配，匹配时用 p[j + 1] 定位到实际字符并和母串进行比较

    ​

  • Q5：KMP匹配成功后，为什么不直接将 j 置为 0，而是继续使用ne数组跳转

  • A5：因为当前匹配成功的串内可能包含下一个可匹配成功的串的一部分，如果直接使 j 为 0，就会跳过这一部分，可能导致情况丢失
